🧪 Synthetic anomaly (preview)

Lightweight synthetic endpoints — no Isolation Forest / Optuna / PyTorch here. Pick a CSV independent of the pipeline CSV in the card below (or reuse the same file). POST /synthetic-preview shows the first N rows before vs after, cell diffs, and injected row indices. POST /synthetic-export downloads the full perturbed CSV for Run Analysis in the pipeline card below or offline.

How injection works (server-side)
  • spike_single — Randomly selects k existing rows (k from contamination). On those rows only, adds magnitude_in_std × (std of the chosen column) to one numeric column (first numeric column if you leave “Single column” empty). Non-numeric columns are unchanged.
  • joint_shift — Same random row mask; adds the same magnitude rule to each selected numeric column (or only the columns listed in “Columns list”). Useful for coordinated multi-sensor bursts.
  • scale_burst — Same row mask; multiplies the chosen numeric columns by scale_factor (default from scenario if omitted). Models multiplicative calibration errors.
Determinism: same random_seed + same file + same parameters ⇒ identical row picks and values. y_true is 1 on perturbed rows, 0 elsewhere — the detector never sees this column; it is only for your evaluation logic.
